REDESIGNING



Learning Adobe XD


For some of these past few case studies, I have been familiarizing myself with Adobe XD, a UI/UX design software developed by the same company as Photoshop and Illustrator. However, surprisingly, I came across a few issues involving accessibility and consistency with the application.



Don't sacrifice accessibility for minimalism.


When I first opened the app, my first observation was that there wasn't any scroll bar. It's nice to know the short cut, space + drag, but by first look, you know you're in for a challenge if you don't know how to move the canvas. So, I added scroll bars and the hand tool which is used in their other applications. Offering these different modes not only contributes to cross-application consistency, but provides accessibility if you can't press space + drag at the same time.



Text Area

Well, maybe that last comment was a bit angry, but the message is loud and clear: users want to incorporate writable text blocks into their designs so their prototypes can be more interactive.

New Feature: Text Area

I also included more options for the text tool, adding their Area Type tool and a new Text Area tool. The Text Area would act a lot like the HTML textarea input, so users would be able to type in the box rather than just making a blank static box. This would make prototyping more interactive and would be great to showcase and test features like comment sections or login pages.

Cosmetic Changes

Other cosmetic changes were made to help with hierarchy and to keep things better connected. For instance, the bottom icons had been used to open pages all the way at the top. To remove the big gap, I made them into tabs above the content, and replaced the icons for text to make it easier to differentiate.

I scooted the Design, Prototype, Share buttons towards the middle so that they'd look more closely related to the canvas rather than the tool bar.

Consistency and Cleaning Up

And, in the right toolbar, I shifted Component to the top to present it higher in the hierarchy, thus making the Align and Grid options more closely related to Transform and Layout.

Another clean up is allocating all the shapes into a single shape tool with options, similar to the tool in Photoshop and Illustrator. This also provides extra options for polygons and star shapes, so you wouldn't have to go back and forth between apps for simple shapes such as these.

"Use 1 control button to active multiple triggers?" "How to apply multiple actions on 1 trigger" "button trigger multiple actions?"

A common question amongst Adobe XD users is how to have one action trigger multiple events. For instance, can clicking a button turn one box blue and another box red? This is currently only possible through an extensive amount of artboards, or by making them all related in a component to do a grand state change... but that can get messy.

I propose a new concept called "Action Chain", which, by using the trigger for the object's state, triggers actions for other components, kind of like an if-statement.

Utilizing a Hierarchical Method

As an example, I wanted to plot what might happen if someone designs a login page. The Text Area tool would have an extra trigger option, "Text", and a specific string will trigger the component to change its state- in this case, "password123" would make the state Pass_True. This also triggers anything listed in Action Chain, so now the Submit button will change its state to Submit_True and, when clicked, the user can visit the next page. Otherwise, the Submit button will be Submit_Default or Submit_False, preventing the option of going forward.

Utilizing a Hierarchical Method

It was quite a feat redesigning a design app from the people who live and breathe design, though providing more accessiblility, interactivity, and consistency throughout their applications would enhance the design experience. Proposing an "Action Chain" feature would take XD to the next step from behaving as a fancy slideshow to becoming an engaging, immersive prototyping tool.
